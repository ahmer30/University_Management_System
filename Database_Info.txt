═══════════════════════════════════════════════════════════════════════════════
SECTION 1 — STUDENT DATA (what the student side needs)
═══════════════════════════════════════════════════════════════════════════════

1.1 STUDENT PROFILE
    - Student ID (e.g. 2021-CS-0042) — primary key, unique
    - Full Name
    - Date of Birth
    - Gender
    - Email (unique)
    - Phone Number
    - Address
    - Profile Picture (store file path as string)
    - Password (hashed)
    - Program / Degree (e.g. BS Computer Science)
    - Enrollment Date
    - Current Semester number
    - Batch / Year (e.g. 2021)
    - Academic Advisor (linked to a teacher)
    - Status (Active / Graduated / Suspended)

1.2 COURSES
    - Course ID (primary key)
    - Course Code (e.g. CS-401)
    - Course Title
    - Description
    - Credits
    - Department
    - Teacher ID (who teaches this course — foreign key to teachers table)
    - Semester (which semester this course belongs to, e.g. Fall 2026)
    - Max Enrollment capacity
    - Schedule / Timetable (days and time, e.g. Mon/Wed 9:00 AM)
    - Room / Location

1.3 STUDENT-COURSE ENROLLMENT
    - Enrollment ID (primary key)
    - Student ID (foreign key)
    - Course ID (foreign key)
    - Enrollment Date
    - Status (Enrolled / Dropped / Completed)
    - Final Grade (letter grade: A, A-, B+, B, etc.)
    - Grade Points (numeric: 4.0, 3.7, 3.3, etc.)

1.4 ASSESSMENTS (per course)
    - Assessment ID (primary key)
    - Course ID (foreign key)
    - Title (e.g. "Midterm Exam", "Assignment 1", "Quiz 2")
    - Type (Quiz / Assignment / Exam / Project / Lab)
    - Total Marks
    - Weightage (% contribution to final grade)
    - Due Date
    - Instructions / Description

1.5 STUDENT ASSESSMENT RESULTS
    - Result ID (primary key)
    - Assessment ID (foreign key)
    - Student ID (foreign key)
    - Marks Obtained
    - Submission Date
    - Status (Pending / Submitted / Graded / Late)
    - Feedback from teacher (text)

1.6 ATTENDANCE
    - Attendance ID (primary key)
    - Student ID (foreign key)
    - Course ID (foreign key)
    - Date
    - Status (Present / Absent / Late / Excused)
    - Marked By (teacher ID, foreign key)

1.7 GPA / TRANSCRIPT
    - Transcript ID (primary key)
    - Student ID (foreign key)
    - Semester label (e.g. "Fall 2023")
    - Semester GPA
    - CGPA up to that semester
    - Total Credits Completed

1.8 TIMETABLE
    - Timetable ID (primary key)
    - Course ID (foreign key)
    - Day of Week (Monday, Tuesday, etc.)
    - Start Time
    - End Time
    - Room / Hall
    - Semester

1.9 MESSAGES / INBOX
    - Message ID (primary key)
    - Sender ID + Sender Type (student or teacher)
    - Receiver ID + Receiver Type (student or teacher)
    - Subject
    - Body (text)
    - Sent At (timestamp)
    - Is Read (boolean)

1.10 NOTIFICATIONS
    - Notification ID (primary key)
    - User ID (student or teacher)
    - User Type
    - Title
    - Body
    - Is Read (boolean)
    - Created At (timestamp)

1.11 FINANCIAL / FEE RECORDS
    - Fee ID (primary key)
    - Student ID (foreign key)
    - Semester
    - Total Fee Amount
    - Amount Paid
    - Balance Due
    - Payment Date
    - Status (Paid / Partial / Unpaid)
    - Scholarship Amount (if any)
    - Scholarship Name (if any)

1.12 ACHIEVEMENTS / BADGES
    - Achievement ID (primary key)
    - Student ID (foreign key)
    - Title (e.g. "Perfect Attendance", "Top 5%", "Dean's List")
    - Description
    - Icon / Emoji
    - Date Awarded

SECTION 2 — TEACHER DATA (what the teacher side needs)
═══════════════════════════════════════════════════════════════════════════════

2.1 TEACHER PROFILE
    - Teacher ID (primary key, e.g. TCH-001)
    - Full Name
    - Email (unique)
    - Password (hashed)
    - Phone Number
    - Department
    - Designation (Professor / Associate Professor / Lecturer / Dr.)
    - Office Location
    - Office Hours (text description)
    - Profile Picture (file path)
    - Date Joined
    - Status (Active / On Leave / Retired)

2.2 COURSES TAUGHT
    - Already covered by the COURSES table above (Teacher ID is a foreign key there)
    - A teacher can teach multiple courses per semester

2.3 RESOURCES (uploaded by teacher per course)
    - Resource ID (primary key)
    - Course ID (foreign key)
    - Teacher ID (foreign key)
    - Title
    - File Path / URL
    - File Type (PDF / Video / Slide / Link)
    - File Size
    - Uploaded At (timestamp)
    - Description

2.4 ANNOUNCEMENTS (per course)
    - Announcement ID (primary key)
    - Course ID (foreign key)
    - Teacher ID (foreign key)
    - Title
    - Body (text)
    - Posted At (timestamp)
    - Is Pinned (boolean)

═══════════════════════════════════════════════════════════════════════════════

═══════════════════════════════════════════════════════════════════════════════

    - Department ID (primary key)
    - Department Name (e.g. Computer Science, Mathematics)

3.2 PROGRAMS / DEGREES
    - Program ID (primary key)
    - Program Name (e.g. BS Computer Science, BS Mathematics)
    - Department ID (foreign key)
    - Duration (years)
    - Total Credits Required

3.3 SEMESTERS
    - Semester ID (primary key)
    - Label (e.g. Fall 2026)
    - Start Date
    - End Date
    - Is Current (boolean)

SECTION 5 — TECHNICAL CONSTRAINTS
═══════════════════════════════════════════════════════════════════════════════

- MySQL version: 8.0 or above
- All passwords must be stored as VARCHAR(255) for bcrypt hashed values
- Use TIMESTAMP for created_at / updated_at fields with DEFAULT CURRENT_TIMESTAMP
- Use ENUM types where there are fixed choices (status fields, day of week, etc.)
- All foreign keys must have ON DELETE CASCADE or ON DELETE SET NULL as appropriate
- Use consistent snake_case naming for all table names and column names
- Do NOT use spaces in column or table names
- The database name is: ums_db
